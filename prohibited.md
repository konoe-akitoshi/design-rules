# Prohibited Patterns（禁止パターン）

これは **デザインガイドライン（[`guidelines/`](./guidelines/README.md)）とは別の、AIへの対応レイヤー**。
デザイン原則は guidelines/ にあり、ここには「**AI生成が外しがちな具体パターンの禁止形**」だけを置く。製作・検証では guidelines と併せて使う。
違反を検出した場合は、右列の代替パターンに置き換えること。

## 構造・骨格（最重要）

配色や題材を変えても「生成された／どこかで見た」印象を最も強く発するのは**レイアウトの骨格**。

| 禁止パターン | 理由 | 代替 |
|-------------|------|------|
| 汎用ランディング骨格（ナビ→eyebrow＋大見出し＋2ボタン＋右図版のヒーロー→等分3カラム特徴→中央引用→3カラムフッター） | 中をいくら抑制しても「定番の型」を発する。生成物の最大のサイン | 題材の native な様式（図面・目録・票・台帳・記事…）から構造を導く（[guidelines/01](./guidelines/01-foundations.md)「題材から構造を導く」） |
| eyebrow（見出し脇の小さい大文字タグライン）の常用 | 情報を運ばない定番装飾 | 情報を運ぶラベルだけを置く |
| 等分3カラムの特徴グリッドを既定にする | テンプレ感の典型 | 内容の関係から列を決める。等分は内容が等価なときだけ |
| 中央寄せの大型引用＋著名を飾りで置く | 定番装飾 | 引用は情報として必要なときだけ、本文の流れに沿わせる |
| 装飾的な `01 / 02 …` の連番見出し（順序が無いのに付ける） | 意味のない構造装飾 | 連番は実際の手順・順序・**索引**があるときだけ可（例: 工程ステップ、編集サイトの章/作品番号 `001…`） |

## スペーシング

| 禁止パターン | 理由 | 代替 |
|-------------|------|------|
| 4の倍数以外のスペーシング値 | 4pt/8ptグリッドから逸脱 | `4px`, `8px`, `16px`, `32px` 等の4の倍数を使用 |
| `padding: 16px`（縦横同値の正方形パディング） | 視覚的に不自然。人間の視覚は横に広い余白を求める | 非対称パディング `padding: 16px 24px`（1:1.5比率）を使用 |
| 縦横比 1:4 以上の極端な非対称パディング | 読みにくさを生む | 1:1.2〜1:2 の範囲に収める |
| 全要素に機械的な等間隔 `margin-bottom: 16px` | Visual Rhythmの欠如 | 見出し下は広め(20px)、段落間は控えめ(12px)、セクション間は明確に(32px) |

## 角丸

| 禁止パターン | 理由 | 代替 |
|-------------|------|------|
| 子要素の角丸 ≥ 親要素の角丸 | 入れ子の同心円理論に反する | `R_inner = R_outer - Padding`（最小2px） |
| `rounded-full` の装飾的乱用 | 過度な丸みは安っぽい印象を与える | 黄金比システム（16px/10px/6px/4px/2px）から選択 |
| 黄金比階層を無視したランダムな角丸値 | 視覚的一貫性の欠如 | Level 0〜4（16px→10px→6px→4px→2px）を使用 |

## 色彩・コントラスト

| 禁止パターン | 理由 | 代替 |
|-------------|------|------|
| 通常テキストのコントラスト比 4.5:1 未満 | WCAG違反 | 4.5:1 以上を確保 |
| 色だけで状態（成功/失敗/警告）を伝える | WCAG 1.4.1 違反・色覚多様性への配慮不足 | アイコン・ラベル・テキストを必ず併用 |
| 補色（赤×緑など）を文字×背景で直接使用 | 色覚多様性に弱い組み合わせ | コントラスト比を確保した上で、モノクロマティックまたは明度差で区別 |
| 赤×緑、青×紫、黄×白の組み合わせ | 色覚多様性に弱い | 明度差を十分に確保した代替色を使用 |
| 1画面に補色配色を3箇所以上使用 | 視覚的に騒がしい | CTA・警告に1〜2箇所のみ |
| 類似色同士で本文テキスト×背景を構成 | コントラスト不足 | WCAG 4.5:1 以上を満たす配色に |

## タッチターゲット

| 禁止パターン | 理由 | 代替 |
|-------------|------|------|
| 主要操作ボタンのタッチ領域 48px 未満 | Fitt's Law・Material Design違反 | 最小 48px × 48px を確保 |
| タッチターゲット間隔 8px 未満 | 誤タップの原因 | `gap: 8px` 以上 |
| モバイルで主要CTAを画面上部に配置 | 親指ゾーンの到達困難領域 | 画面下部の自然ゾーンに配置 |
| セーフエリアインセット未適用 | ベゼルレス端末でのコーナークリッピング | `env(safe-area-inset-*)` を適用 |
| フッター/本文内の密なリンク列に 48/44px を機械適用 | 実サイトでは非現実的（実物は ~36–40px） | 主要48・二次44。**三次（フッター・インライン）は WCAG 2.5.8 の 24px 下限まで可**。行間隔を確保して誤タップを防ぐ |

## 錯視補正

| 禁止パターン | 理由 | 代替 |
|-------------|------|------|
| 三角形アイコンをバウンディングボックス中心に配置 | Triangle-Bisection Illusion | 重心方向へ H/6（約16.7%）オフセット |
| 円形アイコンを正方形と同じサイズで配置 | 円は面積が少なく小さく見える | 3%〜5% 拡大（オーバーシュート） |
| ボタンテキストの line-height のみで垂直中央配置 | Cap Height とディセンダーのズレ | `translateY(-1px)` 等で補正 |

## AI生成パターンの排除

| 禁止パターン | 理由 | 代替 |
|-------------|------|------|
| `shadow-lg`, `shadow-xl` 等の強いシャドウ | AI生成UI特有の過剰装飾 | `shadow-sm` または `shadow` で十分 |
| グラデーション背景の多用 | AI生成UI特有の過剰装飾 | ソリッドカラーを基本とする |
| 絵文字の多用 | AI生成UI特有の装飾 | 必要最小限に抑える。UIアイコンにはSVGを使う |
| カード上部のカラーバー（`border-t-4 border-blue-500` 等） | AI生成パターンの典型 | `border rounded-lg` で全周ボーダー、または装飾なし |
| `text-black` の直接使用 | コントラストが強すぎて不自然 | `text-slate-900` や `text-gray-900` を使用 |
| 装飾過多なヒーローセクション | AIは派手にしがち | コンテンツファーストのミニマルなレイアウト |
| 1ページに3色以上のアクセントカラー | 統一感の欠如 | プライマリ1色 + セマンティックカラー（赤/緑/黄）のみ |
