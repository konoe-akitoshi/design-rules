# Human-Centered Systems Thinking

## 概要

Human-Centered Systems Thinkingは、システムを人間抜きの機械的構造としてではなく、人間の認知、判断、操作、失敗、学習を含むものとして捉える考え方。

このデザインシステムでは、Systems Thinkingを純粋な工学思想で終わらせず、人間中心設計と接続するための核として扱う。

## 背景

複雑なシステムを設計する時、インフラ、データ、アルゴリズム、画面だけを見ても十分ではない。
実際のシステムには、ユーザー、管理者、保守者、チーム、組織、習慣、誤解、例外対応が含まれる。

Human-Centered Systems Thinkingは、こうした人間の行為をシステムの外側ではなく内側に置く。

## 中心的な考え方

```text
System = infrastructure + information + interface + human action
```

この式は、システムを構成するものに人間の判断と行為を含めるための考え方。
ユーザーは外部の観察者ではなく、システムの振る舞いを変える参加者である。

## 代表的な概念

### Human-in-the-loop

自動化されたシステムの中に、人間の判断や確認を組み込む考え方。

### Cognitive Work

ユーザーが頭の中で行う理解、比較、判断、記憶、推論。
UIはこの負荷を減らすだけでなく、必要な判断を支える必要がある。

### Situated Action

行為は抽象的な計画だけでなく、その場の状況に応じて行われる。
UIは固定的な理想フローだけでなく、例外や中断にも耐える必要がある。

## デザインにおける意味

この思想は、UIをユーザーの作業環境の一部として見る。
ユーザーは仕様書通りに動く存在ではなく、状況を読み、迷い、確認し、時に間違える。

そのため、良いUIは次を支える。

* 現在の状況を理解する
* 判断に必要な情報へ到達する
* 不確実性を扱う
* 失敗や見落としから回復する
* 操作結果を理解する

## このデザインシステムとの関係

このデザインシステムでは、Human-Centered Systems Thinkingを「システム思考」と「人間中心設計」の橋渡しとして扱う。
これにより、複雑な構造をただ正しく整理するだけでなく、人間が自然に読める構造にするという方向性が生まれる。
