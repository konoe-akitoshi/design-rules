# Christopher Alexander

## 概要

Christopher Alexanderは、建築家・理論家。
A Pattern Languageなどを通じて、良い設計には繰り返し使えるパターンがあるという考え方を広めた。

このデザインシステムでは、Alexanderを「再利用可能な設計パターン」と「人間にとって自然な構造」の参照点として扱う。

## 背景

Alexanderは建築と都市計画の文脈で、生活に根ざした設計パターンを研究した。
A Pattern Languageは、建築だけでなく、ソフトウェアのDesign Patternsにも影響を与えた。

彼の関心は、設計を抽象的な形式ではなく、人間の行動や生活の中で繰り返し現れる構造として捉えることにあった。

## 中心的な考え方

* 良い設計には再利用可能なパターンがある
* パターンは文脈と問題と解決の関係として現れる
* 人間にとって自然な構造には反復性がある
* 設計は部分の寄せ集めではなく、全体の質を作る

## 代表的な概念

### Pattern

繰り返し現れる問題と、それに対する解決の型。
単なるテンプレートではなく、文脈と意味を含む。

### Pattern Language

複数のパターンが関係し合い、設計全体を構成する体系。

### Quality Without a Name

明確に定義しにくいが、人間にとって生き生きして自然に感じられる設計の質。

## デザインにおける意味

Alexanderの思想は、UIを単発の画面としてではなく、繰り返し現れる構造の集まりとして見る視点を与える。
良いデザインシステムは、見た目の部品集ではなく、問題と解決のパターン集でもある。

UIでは、次のような問いにつながる。

* この画面構造は他でも再利用できるか
* 例外をその場しのぎの見た目で処理していないか
* ユーザーの行動パターンに合っているか
* コンポーネントは単なる部品ではなく意味を持っているか

## このデザインシステムとの関係

このデザインシステムでは、Alexanderは設計ルールの再利用性を支える思想として扱う。
RamsやTufteが表現や情報密度を方向づけるなら、Alexanderはそれらを繰り返し使えるパターンにする視点を与える。
