# Linear

## 概要

Linearは、近年の開発者向けSaaSにおいて、速度、静かな視覚表現、自然な操作感、情報整理の強さで参照されるプロダクト。
このページでは企業や製品紹介としてではなく、現代的な開発者向けUIの思想的参照点として扱う。

このデザインシステムでは、Linearを「Apple的な質感を持ちながら、作業道具として速いUI」の参照点として扱う。

## 背景

開発者向けSaaSでは、見た目の派手さより、反復作業の速さ、検索性、ショートカット、状態把握、チーム内の情報共有が重要になる。
Linearは、そうした作業道具としてのUIを、静かで洗練された視覚表現と結びつけている。

## 中心的な考え方

* 操作が速い
* 視覚表現が静か
* 状態が読みやすい
* アニメーションが自然
* エンジニアの作業密度に耐える

## 代表的な概念

### Speed

操作レスポンス、画面遷移、検索、ショートカットなどを含む体験の速さ。

### Quiet Interface

過剰な装飾や色を避け、作業対象を前に出す表現。

### Keyboard-first Workflows

マウス操作だけでなく、キーボードによる高速な操作を重視する作業観。

### Modern SaaS Craft

細部の動き、余白、タイポグラフィ、状態表示が統合された現代的なUI品質。

## デザインにおける意味

Linearは、単に見た目を真似る対象ではない。
重要なのは、視覚表現と操作速度が矛盾していないこと。
美しいが遅い、静かだが情報が不足している、密度が高いが読めない、という状態を避けるための参照点になる。

UIでは、次のような問いにつながる。

* 反復操作が速いか
* 状態をすぐ読めるか
* 動きが意味を持っているか
* 視覚表現が作業を邪魔していないか
* 熟練者にとっても効率的か

## このデザインシステムとの関係

このデザインシステムでは、LinearをVisual DesignとInteraction Designの接続点として扱う。
Ramsの抑制、Raskinの操作速度、Calm Technologyの静けさを、現代のSaaS体験として接続する参照点である。
