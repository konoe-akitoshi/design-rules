## 錯視補正の原理

### 基本原則

**「数値的に正しい（Geometrically Correct）」≠「見た目に正しい（Optically Correct）」**

人間の視覚システムは、形状の面積、色、周囲の余白、そして重力的なバランスを独自のバイアスで解釈します。厳密な幾何学的配置はしばしば「ズレて」知覚されます。

### 視覚的錯覚が発生する理由

1. **脳の相対認識**
   - 絶対値ではなく、周囲との比較で物を認識
   - 形状の「重心」や「面積の分布」を無意識に探る

2. **曲線と直線の知覚差**
   - 垂直・水平のラインは強く認識される
   - 曲線は「内側に引っ込んでいる」と解釈される傾向

3. **重力と方向性のバイアス**
   - 人間は下方向への重力を感じているため、下方向の要素を「重い」と認識
   - 上方向は「軽い」「浮いている」と感じる

## 三角形の重心補正

### 再生ボタンのパラドックス

円形ボタンの中に三角形（再生アイコンなど）を配置する場合、**幾何学的な中心（Bounding Boxの中心）に配置すると、三角形は重心の偏りにより左に寄って見えます**（右向き三角形の場合）。

これは「Triangle-Bisection Illusion」と呼ばれる現象に関連しています。

### 重心補正の数式

三角形の視覚的な中心は、バウンディングボックスの中心ではなく、**幾何学的重心（Centroid）**に近い位置にあります。

**三角形の重心位置：**
- 各頂点から対辺の中点へ引いた線（中線）の交点
- 底辺から高さの **1/3** の位置

**バウンディングボックスの中心：**
- 高さの **1/2** の位置

したがって、右向きの三角形を視覚的に中央に配置するには、バウンディングボックスの中心から右方向へ以下のオフセット量だけずらす必要があります。

$$
\text{Offset} = \text{Center}_{box} - \text{Center}_{gravity} = \frac{H}{2} - \frac{H}{3} = \frac{H}{6}
$$

ここで $H$ は三角形の高さ（底辺からの距離）。

**結論: 右向きの三角形の場合、幅の約16.7% (1/6) だけ右にずらす**

### 実装例

```css
/* 円形再生ボタン */
.play-button {
  width: 64px;
  height: 64px;
  border-radius: 50%;
  display: flex;
  align-items: center;
  justify-content: center;
  position: relative;
}

/* 三角形アイコン（SVG） */
.play-button svg {
  /* 重心補正: 右に約16.7%移動 */
  transform: translateX(10%); /* 視覚調整版（実測値） */

  /* または厳密に */
  /* transform: translateX(calc(100% / 6)); */
}
```

```html
<!-- SVG三角形 -->
<button class="play-button">
  <svg width="24" height="24" viewBox="0 0 24 24">
    <polygon points="8,5 19,12 8,19" fill="currentColor"/>
  </svg>
</button>
```

### 方向別の補正係数

| アイコンの方向 | オフセット方向 | 係数 | 補正値 |
|-------------|-------------|-----|--------|
| 右向き（▶） | 右へ | +1/6 | +16.7% |
| 左向き（◀） | 左へ | -1/6 | -16.7% |
| 上向き（▲） | 上へ | -1/6 | -16.7% |
| 下向き（▼） | 下へ | +1/6 | +16.7% |

## オーバーシュート（円と四角の面積等価）

### 円と正方形の知覚差

正方形と円を同じ高さ（直径＝一辺）で並べると、**円の方が小さく見えます**。

**理由：**
1. 円は四隅の面積が欠落しており、総面積が小さい
2. 人間の目は垂直・水平のラインより曲線を「内側に引っ込んでいる」と解釈

### 面積等価に基づく補正係数

視覚的なバランスを取るためには、円の直径を正方形の一辺よりも大きくし、面積を近づける必要があります。

**数学的計算：**

正方形の一辺を $a$、円の半径を $r$ とすると：

- 正方形の面積: $S_{sq} = a^2$
- 円の面積: $S_{circ} = \pi r^2$

両者の面積を等しくするには：

$$
a^2 = \pi r^2 \quad \Rightarrow \quad r = \frac{a}{\sqrt{\pi}} \approx 0.564a
$$

直径 $D = 2r \approx 1.128a$

**つまり、数学的に面積を等しくするには、円の直径を正方形の一辺より約12.8%大きくする必要があります。**

### 実用的な補正値

しかし、実際には面積だけでなく「高さ」もサイズ認知の強い手がかりとなるため、完全な面積等価（約13%拡大）では円が大きすぎると感じられます。

**経験的な補正値: 3%〜5% 程度の拡大（オーバーシュート）が最適解**

### ルール化

**アイコンセットやタイポグラフィにおける補正：**

1. **円形要素（O, C, 円形アイコン）**
   - 基準線（ベースラインやキャップハイト）から上下に **1%〜3%** 程度はみ出させる

2. **先端が尖った要素（A, V, 三角形）**
   - 同様に **1%〜3%** はみ出させる

3. **具体例**
   - 40pxの正方形アイコンとバランスを取る円形アイコンは、**41px〜42px** 程度の直径で設計

### 実装例

```css
/* アイコングリッドシステム */
:root {
  --icon-size-base: 40px;
  --icon-size-circular: 42px; /* 5%拡大 */
}

/* 正方形アイコン */
.icon-square {
  width: var(--icon-size-base);
  height: var(--icon-size-base);
}

/* 円形アイコン（オーバーシュート補正） */
.icon-circle {
  width: var(--icon-size-circular);
  height: var(--icon-size-circular);
  border-radius: 50%;
}

/* SVGアイコンでの補正 */
.icon-o {
  /* 円形文字は垂直方向に3%拡張 */
  transform: scaleY(1.03);
}
```

## テキストとグリッドの視覚的整列

### オプティカルマージン（Optical Margin）

角丸のコンテナ内にテキストを配置する場合、あるいは角丸ボタンの横にテキストを配置する場合、単純な左揃えではテキストの行頭が揃っていないように見えることがあります。

### 行頭約物とオプティカルマージン

特に左端が以下の形状で始まる場合、視覚的な凹みが発生します：

- **文字**: T, V, Y, W, A など（左側に空間がある字形）
- **約物**: "（引用符）、'（アポストロフィ）など

**原因：**
文字の物理的な境界ボックス（Bounding Box）の左端をグリッドラインに合わせると、文字の主要部分（ステム）が右に寄っているため、行頭が凹んで見える。

### ルール化

**見出しや主要なテキストブロックにおいて、行頭に約物や特定の文字が来る場合：**

1. グリッドラインより左側（マージン領域）に**マイナスのインデント（ネガティブマージン）**を持たせる
2. フォントのグリフメトリクスに基づいた補正を行う

### 実装例

```css
/* 見出しのオプティカルマージン */
h1, h2, h3 {
  /* 行頭が特定の文字の場合に適用 */
  hanging-punctuation: first last; /* CSS3（対応ブラウザ限定） */
}

/* 手動補正 */
.heading-with-quote::first-letter {
  margin-left: -0.3em; /* 引用符などの場合 */
}

/* JavaScript + CSS Variables */
.heading[data-first-char="T"],
.heading[data-first-char="V"],
.heading[data-first-char="W"] {
  transform: translateX(-0.05em); /* 5%左にずらす */
}
```

### 補正係数ガイドライン

| 文字/約物 | 補正量 | 説明 |
|----------|-------|------|
| "（引用符） | -0.3em〜-0.5em | 強い補正 |
| T, V, W | -0.05em〜-0.1em | 軽度の補正 |
| A, Y | -0.03em〜-0.05em | 微細な補正 |
| O, C | +0.02em〜+0.03em | 曲線のはみ出し |

## ボタン内テキストのキャップハイト補正

### 問題: ボタンテキストが下に寄って見える

ボタンの中にテキストを垂直配置する際、CSSの `line-height` や flexbox の `center` で配置すると、多くの欧文フォントでは**テキストが「下に寄って」見えます**。

**原因：**
- 大文字の高さ（Cap Height）よりも、小文字のディセンダー（g, j, p, q, yの下に伸びる部分）を含めたラインハイト全体でセンタリングされるため
- フォントのベースライン位置が視覚的な中心とずれている

### キャップハイト基準のセンタリング

**ルール化：**

ボタンのラベルなど、大文字中心またはディセンダーを含まないテキストの場合、**Cap Height を基準にセンタリングを行う**。

### 実装例

```css
/* 方法1: パディング調整 */
.button {
  padding: 12px 24px 14px 24px; /* 下パディングを2px多く */
}

/* 方法2: Transform補正 */
.button-text {
  transform: translateY(-1px); /* 1px上にずらす */
}

/* 方法3: Flexbox微調整 */
.button {
  display: flex;
  align-items: center;
  justify-content: center;
  line-height: 1; /* ラインハイトを詰める */
}

.button-text {
  /* フォント固有のメトリクス調整 */
  --font-cap-height-ratio: 0.7; /* フォントによって異なる */
  margin-top: calc((1 - var(--font-cap-height-ratio)) * -0.5em);
}
```

### フォント別の補正係数

| フォント | Cap Height / Em | 推奨補正 | 説明 |
|---------|----------------|---------|------|
| Helvetica | 0.72 | -1px〜-2px | 標準的な補正 |
| Arial | 0.72 | -1px〜-2px | Helveticaと同等 |
| Roboto | 0.73 | -1px | やや控えめ |
| Inter | 0.72 | -1px〜-2px | 標準的な補正 |
| SF Pro (Apple) | 0.71 | -2px | やや強めの補正 |
| Noto Sans JP | 0.88 | 0px | 日本語フォントは補正不要の場合が多い |

### 自動計算（高度な実装）

```javascript
// フォントメトリクスの取得
function getCapHeightRatio(fontFamily) {
  const canvas = document.createElement('canvas');
  const ctx = canvas.getContext('2d');
  ctx.font = `100px ${fontFamily}`;

  const metrics = ctx.measureText('H');
  const capHeight = metrics.actualBoundingBoxAscent;
  const fontSize = 100;

  return capHeight / fontSize;
}

// CSS変数に設定
const ratio = getCapHeightRatio('Inter');
document.documentElement.style.setProperty('--font-cap-height-ratio', ratio);
```

## 実装ガイドライン

### 錯視補正チェックリスト

**実装前のチェック項目：**

- [ ] 三角形アイコンは重心位置で配置されているか？
- [ ] 円形アイコンは3%〜5%のオーバーシュートを持っているか？
- [ ] 行頭の約物や特殊文字にオプティカルマージンが適用されているか？
- [ ] ボタンテキストはキャップハイトを基準にセンタリングされているか？
- [ ] 異なるフォントサイズでも補正が機能しているか？

### 段階的実装アプローチ

**優先順位：**

1. **高優先度（必須）**
   - 再生ボタンの三角形補正
   - ボタンテキストの垂直センタリング補正

2. **中優先度（推奨）**
   - 円形アイコンのオーバーシュート
   - 見出しのオプティカルマージン（引用符など）

3. **低優先度（オプション）**
   - 細かな文字形状の補正（T, V, W等）
   - フォント固有のメトリクス調整

### パフォーマンス考慮事項

**静的補正 vs 動的補正：**

| 手法 | パフォーマンス | 精度 | 推奨用途 |
|-----|------------|-----|----------|
| 固定値補正（px, %） | ★★★★★ | ★★★☆☆ | 大量の要素、標準的なケース |
| em/rem相対補正 | ★★★★☆ | ★★★★☆ | レスポンシブデザイン |
| JavaScript計算 | ★★☆☆☆ | ★★★★★ | プレミアムUI、特殊フォント |

### CSS変数による補正システム

```css
:root {
  /* 錯視補正係数 */
  --optical-triangle-offset: 10%; /* 三角形の重心補正 */
  --optical-circle-overshoot: 1.05; /* 円形のオーバーシュート */
  --optical-cap-height-shift: -1px; /* ボタンテキストの垂直補正 */
  --optical-margin-quote: -0.3em; /* 引用符のマージン */

  /* フォント固有の係数 */
  --font-cap-height-ratio: 0.72; /* デフォルト: Helvetica/Arial */
}

/* 適用例 */
.play-icon {
  transform: translateX(var(--optical-triangle-offset));
}

.circular-icon {
  width: calc(var(--icon-size) * var(--optical-circle-overshoot));
  height: calc(var(--icon-size) * var(--optical-circle-overshoot));
}

.button-text {
  transform: translateY(var(--optical-cap-height-shift));
}

.quote-heading::first-letter {
  margin-left: var(--optical-margin-quote);
}
```

## まとめ：人間中心の設計思想

錯視補正は、**「数学的完璧性」より「視覚的完璧性」を優先する人間中心設計**の実践です。

### 核心原則

1. **幾何学的中心 ≠ 視覚的中心**
   - 重心、面積、方向性のバイアスを考慮

2. **測定可能な補正係数**
   - 経験則を数式化し、再現可能なルールに

3. **段階的適用**
   - 優先度に応じて実装範囲を調整

4. **継続的な視覚テスト**
   - 数式はあくまで出発点、最終判断は目で行う

**目的: ユーザーが無意識に感じる「心地よさ」「違和感のなさ」の実現**

この微細な配慮の積み重ねが、プロフェッショナルなデザインとアマチュアのデザインを分ける決定的な差となります。
