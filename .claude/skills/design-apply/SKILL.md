# Design Apply — DS準拠でUIを生成

デザインルールに準拠したUIコードを生成する。

## 必須: ルールの読み込み

作業前に以下を必ず読み込むこと:

1. `${CLAUDE_SKILL_DIR}/../../../CLAUDE.md` — クイックリファレンス（まずこれを読む）
2. `${CLAUDE_SKILL_DIR}/../../../guidelines/README.md` — 決定順（見た目から始めない）と章索引
3. `${CLAUDE_SKILL_DIR}/../../../tokens.json` — トークン定義（具体的な値はここが正）
4. `${CLAUDE_SKILL_DIR}/../../../guidelines/values/color-system.md` — 配色の実値（`--c-*`）
5. `${CLAUDE_SKILL_DIR}/../../../prohibited.md` — 禁止パターン（生成後にセルフチェック）

タスクに応じて該当 ガイドライン 章を読む（例: 状態UI→03/08、レイアウト→04、配色→05、タイポ→06、抑制→07）。
**ガイドライン の決定順（目的→構造→操作→レイアウト→造形→抑制→検証）に従い、見た目から始めない。**

### ジャンルでA/Bを判別する（重要）
- **ブランドサイト・記事・LP・印刷物** → **A章のみ**（02/04/05/06/07/09/10 ＋ 01の一般原則）。
  - 01の「**題材から構造を導く・汎用テンプレ骨格は禁止**」を必ず適用。eyebrow＋大見出し＋2ボタン＋右図版のヒーロー、等分3カラム、中央引用、装飾的01/02番号を既定で使わない。題材の native な様式（図面・目録・票・台帳・記事…）から構造を立てる。
- **ダッシュボード・管理・監視・運用ツール（動的な状態/操作）** → **A＋B**（03のCybernetics・08の例外状態を追加）。
- 判別が曖昧なら「観測する状態・操作・制御があるか？」で決める。無ければ A のみ。詳細は [guidelines/README 適用範囲](../../../guidelines/README.md)。

## 生成ルール

### スペーシング
- 4pt/8pt グリッドに準拠（4の倍数のみ使用）
- パディングは必ず非対称にする
  - カード: `py-4 px-6`
  - ボタン: `py-3 px-5`
  - タグ: `py-1 px-3`
  - フォーム: `py-2 px-4`
- Visual Rhythm を意識（見出し下は広め、段落間は控えめ、セクション間は明確に）

### 角丸
- 黄金比システムを使用: 16px / 10px / 6px / 4px / 2px
- 入れ子では R_inner = R_outer - Padding（最小2px）
- 親 > 子 > 孫 の関係を厳守

### 色彩
- `text-black` は使わない → `text-slate-900`
- ハードコード色は使わない
- WCAG コントラスト比を意識（通常テキスト 4.5:1）
- 色だけで状態を伝えない（アイコン・テキスト併用）

### 装飾
- `shadow-sm` のみ使用（`shadow-lg`, `shadow-xl` 禁止）
- グラデーション不使用（ソリッドカラー基本）
- カラーバー（`border-t-4` + 色）不使用
- 絵文字不使用（SVGアイコン推奨）

### タッチターゲット
- ボタン・リンク: 最小 48px × 48px
- インタラクティブ要素間隔: 最小 8px
- モバイルCTA: 画面下部に配置

### 錯視補正
- 三角形アイコン: 尖端方向へ 16.7% オフセット
- 円形アイコン: 3-5% 拡大
- ボタンテキスト: `translateY(-1px)` 補正

## セルフチェック

コード生成後、以下を確認してから出力すること:

1. prohibited.md の禁止パターンに該当する箇所がないか
2. tokens.json のトークン値と一致しているか
3. 非対称パディングが適用されているか
4. 角丸の親子関係が正しいか
5. `shadow-lg`, `text-black`, カラーバーが含まれていないか

違反があれば修正してから出力する。
