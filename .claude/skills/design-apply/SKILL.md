# Design Apply — DS準拠でUIを生成

デザインルールに準拠したUIコードを生成する。

## 必須: ルールの読み込み

作業前に以下を必ず読み込むこと:

1. `${CLAUDE_SKILL_DIR}/../../../CLAUDE.md` — クイックリファレンス（まずこれを読む）
2. `${CLAUDE_SKILL_DIR}/../../../tokens.json` — トークン定義（具体的な値はここが正）
3. `${CLAUDE_SKILL_DIR}/../../../prohibited.md` — 禁止パターン（生成後にセルフチェック）

## タスク

$ARGUMENTS

## 生成ルール

### スペーシング
- 4pt/8pt グリッドに準拠（4の倍数のみ使用）
- パディングは必ず非対称にする
  - カード: `py-4 px-6`
  - ボタン: `py-3 px-5`
  - タグ: `py-1 px-3`
  - フォーム: `py-2 px-4`
- Visual Rhythm を意識（見出し下は広め、段落間は控えめ、セクション間は明確に）

### 角丸
- 黄金比システムを使用: 16px / 10px / 6px / 4px / 2px
- 入れ子では R_inner = R_outer - Padding（最小2px）
- 親 > 子 > 孫 の関係を厳守

### 色彩
- `text-black` は使わない → `text-slate-900`
- ハードコード色は使わない
- WCAG コントラスト比を意識（通常テキスト 4.5:1）
- 色だけで状態を伝えない（アイコン・テキスト併用）

### 装飾
- `shadow-sm` のみ使用（`shadow-lg`, `shadow-xl` 禁止）
- グラデーション不使用（ソリッドカラー基本）
- カラーバー（`border-t-4` + 色）不使用
- 絵文字不使用（SVGアイコン推奨）

### タッチターゲット
- ボタン・リンク: 最小 48px × 48px
- インタラクティブ要素間隔: 最小 8px
- モバイルCTA: 画面下部に配置

### 錯視補正
- 三角形アイコン: 尖端方向へ 16.7% オフセット
- 円形アイコン: 3-5% 拡大
- ボタンテキスト: `translateY(-1px)` 補正

## セルフチェック

コード生成後、以下を確認してから出力すること:

1. prohibited.md の禁止パターンに該当する箇所がないか
2. tokens.json のトークン値と一致しているか
3. 非対称パディングが適用されているか
4. 角丸の親子関係が正しいか
5. `shadow-lg`, `text-black`, カラーバーが含まれていないか

違反があれば修正してから出力する。
