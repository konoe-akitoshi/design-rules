# Web Design Guidelines — 設計原則のQAゲート

指定された UI を、ガイドライン（使えるデザインガイドライン）の設計原則に照らして検査する。
`design-review` が禁止パターンとトークンの**機械的**検出なら、こちらは情報構造・操作・抑制・状態・a11y の**原則レベル**検査。各指摘に「なぜ問題か（理論）」を添える。

## 読み込む

検査前に必ず読む:

1. `${CLAUDE_SKILL_DIR}/../../../guidelines/README.md` — 決定順と章構成
2. `${CLAUDE_SKILL_DIR}/../../../guidelines/01-foundations.md` 〜 `10-accessibility.md` — 各章末の「統合チェックリスト」を検査基準にする
3. `${CLAUDE_SKILL_DIR}/../../../guidelines/README.md` の「優先順位」 — 矛盾時の floor
4. `${CLAUDE_SKILL_DIR}/../../../prohibited.md` — AI生成の禁止パターン（併せて確認）

## 検査対象

$ARGUMENTS

## 検査手順

各 ガイドライン 章の「統合チェックリスト」を順に当てる。最低限の観点:

- **02 情報構造**: 整理軸が単一で表示されているか / 現在地が読めるか / 境界が明示か / 重要度と視覚階層が一致か / chartjunk が無いか / 比較は同一軸か / 判断材料を削っていないか
- **03 操作とFB**: 状態UIに観測・判断・制御・FBが揃うか / signifier・即時FBがあるか / エラーは原因＋回復か / 破壊操作にundoか / 強調が変化対象に絞られ、重大変化は可視か
- **07 抑制と造形**: クロームが内容より目立たないか / 影は弱いか / 根拠なき造形・見せかけの形が無いか / 削れる装飾を削り手がかりは残したか
- **08 状態と耐性**: 空/読込中/エラー/権限/部分 の5状態が設計されているか / 大量・長文で崩れないか
- **10 a11y**: コントラスト / 色のみ依存 / タッチターゲット / focus可視 / アイコンラベル

## 出力フォーマット

```
## Guidelines 検査結果

### サマリー
- Critical（下限違反: 可読性/判断材料/回復/重大変化の可視/a11y）: X件
- Warning（原則違反）: Y件
- Info（改善提案）: Z件

### 詳細

#### [Critical] 現在地が読めない（02 情報構造 / Legibility）
- 箇所: ファイル:行
- なぜ問題か（理論）: 読める構造でなければ操作対象にならない。現在地・関係・状態が判別できることは美しさより前の下限。
- 修正: active状態 / パンくず / セクション見出しを追加する。

...
```

重大度は guidelines/README「優先順位」の floor（可読性 / 判断材料 / 回復可能性 / 重大変化の可視性 / 実用性）違反を Critical とする。
指摘は必ず「なぜ問題か（理論）」と「該当ガイドライン章」を添えること。
