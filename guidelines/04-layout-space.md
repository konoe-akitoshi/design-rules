# 04 — Layout & Space（レイアウトと余白）

操作できるようになった情報を、空間配置で「関係」と「読み順」へ翻訳する。
関心: **グリッド・余白・近接・読み順・錯視補正**。決定順の段4。配置そのものが情報であり、文字説明の前に読まれる。

各項目は `思想 → その思想を使うための理論 → 使える規則` の3層。値は `tokens.json` を参照。

---

## International Typographic Style（Swiss） — グリッドで関係と階層を示す

**思想**
グリッドで秩序と読み順を作り、サイズ・ウェイト・余白で階層を示す。中央揃えの装飾的対称ではなく、情報の重みと配置で非対称にバランスを取る。

**その思想を使うための理論**
共通の整列軸（列グリッド・ベースライン）を共有する要素は、人の目に「同じ仲間」として読まれる（整列のゲシュタルト）。視線は強い縦横ラインに沿って動くので、整列は見た目の整頓ではなく**読み順そのもの**を作る。階層はサイズ・ウェイト・余白の差で作るのが本筋で、色や装飾に頼ると軸が増えてノイズになる（情報構造の詳細は [02 information-structure](./02-information-structure.md)）。中央揃えは全要素を同じ強さ・同じ軸に揃えてしまい、階層と読み始め位置を消す。だから主たる軸へ左/上揃えで置き、空いた側に視覚的な重さを逃がして非対称にバランスさせる。運用手順は、まず列数（4/8/12列）とベースラインを決め、全要素の端をラインに乗せ、階層を2〜3段に絞って各段でサイズ or ウェイトの差を明確につける。

**使える規則**
- 列グリッド＋ベースラインを定義し、要素の左端・上端をライン上に揃える。
- 階層はサイズ・ウェイト・余白の差で作る（色・装飾・罫線で階層を作らない）。段数は2〜3に絞る。
- 中央揃えは短い単独要素（ヒーロー見出し等）に限定。本文・リスト・フォームは左揃え。
- 中央対称に逃げず、非対称配置でバランスを取る（余白を意味に応じて配分する）。

## Ecological Psychology — 配置が関係を語る（近接と整列）

**思想**
人は要素を孤立した記号ではなく、距離・向き・まとまりとして知覚する。意味は対象単体ではなく**関係**から生まれ、空間配置はそのまま理解に影響する。

**その思想を使うための理論**
近接の原則 — 近いものは一群、離れたものは別群と無意識に読む。これは文字を読む前に成立する前注意的（pre-attentive）処理なので、関係を文字ラベルだけで説明させると二度手間になり認知負荷が上がる。群内の余白 < 群間の余白という段差をはっきりつけないと境界が曖昧になり、ユーザーが構造を頭の中で再構築するコストを払う。整列も同じく軸の共有でまとまりを作る。つまり「ラベルと値が一組」「アイコンとテキストが一組」を間隔で示せば、配置自体が関係を語る。操作対象が自然に分かることも環境（配置）から直接読まれる（操作の mapping/signifier は [03 interaction-feedback](./03-interaction-feedback.md)）。運用は、関連要素どうしを近づけ、無関係な群との間を必ず1段階開けること。

**使える規則**
- 関連要素は近接させ、無関係な要素は離す。群内余白 < 群間余白を最低1段差つける（例: 群内 8px / 群間 24px）。
- ラベルと対象、アイコンとテキスト、見出しと本文は密接配置にして一組と読ませる。
- 整列軸を共有させてまとまりを作る（バラバラな端揃えで群を壊さない）。
- 関係は配置で示し、文字説明だけに頼らない（「※これは○○です」で位置の悪さを補わない）。

## 余白の運用 — グリッド・非対称パディング・Visual Rhythm

**思想**
余白は要素の隙間ではなく構造そのもの。4pt/8ptグリッドで一貫させ、非対称パディングで自然さを、リズムで読みやすさを作る。

**その思想を使うための理論**
人の目は不規則な間隔を「ノイズ」と読み、共通単位の倍数を「秩序」と読む。だから余白は4の倍数（`spacing` トークン）から選び、勝手な値を混ぜない。階層は値の差で表す — 要素内余白 < 要素間余白 < セクション間余白。パディングを縦横同値にすると窮屈で不自然に見えるのは、人の視野が横長で、横方向により多くの余白を要求するため。だから縦 < 横の非対称比にする。Visual Rhythm は等間隔の単調さを崩して読み順を余白で語る手法で、見出しはセクションの入口なので下を広めに、段落間は同一の流れなので控えめに、セクション間は別ブロックなので明確に開ける。これで「どこが塊か」が間隔から読める。運用手順は、まずグリッド単位を適用 → 要素タイプごとに非対称パディング表を当てる → リズムで最終調整。

**使える規則**
- 余白は `spacing` トークンから選ぶ（4 / 8 / 12 / 16 / 24 / 32 / 48 / 64px）。勝手な値を使わない。
- 階層を守る: 要素内余白 < 要素間余白 < セクション間余白。
- パディングは非対称（`asymmetricPadding`）。縦横同値は禁止。
  - カード `16px 24px`（1:1.5）/ ボタン `12px 20px`（1:1.67）/ バナー `32px 48px`（1:1.5）/ タグ `4px 12px`（1:3）/ フォーム `8px 16px`（1:2）
- Visual Rhythm: 見出し下は広め（20px）・段落間は控えめ（12px）・セクション間は明確（32px）。
- タッチターゲット最小（48/44px, `touchTargets`）は余白圧縮より優先（詳細は 10 accessibility *(予定)*）。

## 錯視補正 — 数値的中心ではなく視覚的中心に合わせる

**思想**
「数値的に正しい」≠「見た目に正しい」。人の知覚は重心・面積・重力にバイアスを持つので、配置を数pxずらして視覚的中心に合わせる。

**その思想を使うための理論**
脳は絶対値ではなく重心と面積で位置・大きさを推定する。だから幾何学的に正しい配置がしばしば「ズレて」見える。三角形はバウンディングボックスの中心（高さ1/2）と重心（底辺から1/3）がずれるので、幾何中心に置くと尖端と逆へ寄って見える → 尖端方向へ H/6（≈16.7%）オフセットする。円は四隅が欠けて面積が小さく、曲線が内側に引っ込んで見えるため、正方形と並べると小さく見える → 3〜5%拡大（オーバーシュート）する。ボタンのラベルはディセンダー込みの line-height で中央化されるとキャップの中心が下に寄る → 1px 上げる。テキストの視覚的重量も同様で、太字は黒みが多く近く見えるので下余白を詰め、細字は軽いので開ける。これは構造（前の3節）が整った後の最終ポリッシュであり、トークン値はあくまで出発点、最終判断は目で行う。

**使える規則**
- 三角アイコン（▶等）は尖端方向へ H/6（≈16.7%）オフセット（`triangleCentroid`）。
- 円形アイコンは正方形比で3〜5%拡大（`circleOvershoot` factor 1.05）。
- ボタンラベルは `translateY(-1px)`（`capHeightShift`）。Noto Sans JP 等の日本語フォントは0で可。
- 太字テキスト下余白は標準 ×0.75、軽字テキスト下余白は ×1.25（`opticalSpacing`）。
- 補正値はトークンを出発点に、異なるサイズでも機能するか目で最終確認する。

## 統合 — この章の使える規則（チェックリスト）

1. 列グリッド＋ベースラインを定義し、要素の端をラインに揃えたか
2. 階層をサイズ・ウェイト・余白の差で作ったか（2〜3段に絞り、色・装飾に頼っていないか）
3. 中央揃えに逃げず、非対称でバランスを取ったか
4. 関連要素を近接させ、無関係を離したか（群内余白 < 群間余白を1段差以上）
5. 関係を配置で示したか（文字説明だけで位置の悪さを補っていないか）
6. 余白を `spacing` トークンから選び、要素内 < 要素間 < セクション間の階層を守ったか
7. パディングを非対称にしたか（縦横同値を使っていないか）／Visual Rhythm（見出し下広め・段落間控えめ・セクション間明確）か
8. 三角アイコン・円形アイコン・ボタン文字・太字余白に錯視補正をかけたか

衝突時の優先: 近接による関係の表現(4) は構造の可読性に直結するため、装飾的な等間隔より優先する。タッチターゲット最小(48/44px) は余白圧縮より優先（メタ下限、10 accessibility）。錯視補正(8) は最後のポリッシュで、構造(1〜7) が崩れた箇所には適用しない（順序を守る）。
