# 表示・コンテンツ — コンポーネント詳細

情報そのものを運ぶ容器・図像・構造の部品群。[共通規則](./README.md#どのコンポーネントも継承する共通規則universal)（signifier・focus・基本状態・色役割・タッチ下限・コントラスト）は各仕様で繰り返さない。値は [`tokens.json`](../../tokens.json) / [color-system](../values/color-system.md)。

---

### Card  〔Tile〕

**用途** 単一エンティティ（記事・商品・ユーザー等）の情報を1つの操作単位に束ねる。一覧の反復要素。
**変種** 静的カード / リンクカード（全体が1リンク）/ アクション付き / 水平・垂直 / メディア先頭。
**構成** メディア（任意）＋見出し＋本文・メタ＋アクション（任意）。並び順＝読み順＝重要度。
**寸法** padding `16px 24px`（1:1.5, asymmetricPadding.card）／radius 10px（golden-1, `rounded-lg`）。内側メディアは `R_inner = R_outer − padding`、端まで伸ばす画像は最小2px。カード間 gap は群間 24px（spacing-6）。
**色** 面 `--c-surface`、境界はヘアライン `--c-line`（影で浮かせない）。本文 `--c-ink`／メタ `--c-muted`。`--c-canvas` 上に置く。
**状態** リンク・操作カードのみ hover で面 `surface→sunk`、選択中は地 `--c-accent-sunk`＋境界 `--c-accent`。静的カードは状態を持たない。
**規則** Tufte: カードは「箱」を量産しがち。余白で群が読めるならカード化しない（箱でなくヘアライン・余白で分ける）。`border-t-4` のカラーバー禁止＝全周 `--c-line` か装飾なし。Alexander: 同種エンティティは1つのカードパターンに統一し、項目ごとの one-off レイアウトを作らない。
**a11y** カード全体をリンクにするなら `<a>` で包み、内側にリンクを入れ子にしない。見出しは画面階層に合わせ `<h2>`〜`<h3>`。装飾画像は `alt=""`。
**関連** Stack（内部余白）／List（並び）／Image。

### Table  〔Data table〕

**用途** 同型レコードを行×列で並べ、列方向に比較・走査させる。大量データの観測・ソート・フィルタ。
**変種** 基本 / 行選択（チェックボックス）/ ソート可 / 固定ヘッダ・固定列 / 行展開 / ページネーション付き。
**構成** `caption`＋`thead`（列見出し＋ソートトリガ）＋`tbody`（行）＋（任意）`tfoot` 合計。数値列は右揃え・等幅（mono）。
**寸法** セル padding `8px 12px` 前後（縦<横）／角丸なし（表は罫線で構造化）。行高はタッチ44px以上を確保。
**色** 罫線は水平ヘアライン `--c-line` のみ（縦罫・外枠の箱は引かない）。ヘッダ文字 `--c-muted`、本文 `--c-ink`。ゼブラを使うなら `--c-sunk` で極淡く。
**状態** 行 hover `--c-sunk`／選択行 `--c-accent-sunk`＋左端アクセント／ソート中の列ヘッダを `--c-ink`＋方向矢印で明示。空・読込（スケルトン行）・エラー・権限なし・部分件数の5状態を必ず設計（08）。
**規則** Tufte: data-ink 最大化＝縦罫線・セル塗り・3D影を削り、横ヘアラインと余白で行を分ける（chartjunk 排除）。比較は同一列・同一単位・同一フォーマット（小数桁・通貨を揃える）。Wurman: 現在のソート軸を列ヘッダに表示（どの LATCH 軸で並ぶか）。Alexander: セルの例外表現を one-off にせず Badge 等の既存パターンで。
**a11y** `<table>`＋`<caption>`。列見出し `<th scope="col">`、行見出し `<th scope="row">`。ソートは `aria-sort`。レイアウト目的にテーブルを使わない。
**関連** Pagination／Badge／Checkbox／Empty state。

### List

**用途** 同種の関連項目を1つのまとまりとして縦に並べる。順序・対応の有無で要素型を選ぶ。
**変種** 順不同 `ul` / 順序 `ol`（手順・順位）/ 定義 `dl`（用語↔説明）/ リンクリスト / アクション付き行。
**構成** 各項目＝先頭マーカー・アイコン（任意）＋主テキスト＋副次メタ（任意）＋末尾アクション（任意）。
**寸法** 項目内 padding は縦<横。項目間は群内余白（8px, spacing-2）、リスト間は群間（24px）。操作可能な行は最小高44px。
**色** 区切りは余白を基本に、必要時のみ項目間ヘアライン `--c-line`。主 `--c-ink`／メタ `--c-muted`。
**状態** 操作可能な行のみ hover `--c-sunk`、選択中 `--c-accent-sunk`。
**規則** Wurman（02）: 並び順は単一の LATCH 軸（時系列・五十音・重要度…）で、軸を見出し・ラベルで明示。Tufte: 1行ずつ箱で囲まない、罫線より余白で分離。Alexander: 行レイアウトを1パターンに統一し項目ごとに変えない。意味が「順序あり」なら必ず `ol`。
**a11y** リスト構造は `ul/ol/dl` のネイティブ要素で（`div` の羅列にしない＝支援技術に件数・項目が伝わる）。定義は `dl>dt+dd`。
**関連** Card／Table／Stack／Tree view。

### Accordion  〔Disclosure〕

**用途** 見出しで内容を開閉し、縦の長さを畳んで一覧性を上げる。二次情報・FAQ・設定群。
**変種** 単一開閉（Disclosure）/ 複数同時開 / 排他（1つだけ開く）/ アイコン＋見出し行。
**構成** トリガ（見出しボタン＋開閉インジケータ）＋折り畳みパネル。インジケータは回転シェブロン等で状態を示す。
**寸法** トリガ行は最小高44px、padding 縦<横。パネル padding は `16px 24px` 系。外枠角丸 10px、内側は `R_inner = R_outer − padding`（最小2px）。
**色** 面 `--c-surface`、項目間ヘアライン `--c-line`。見出し `--c-ink`。展開中も地色を変えない（静かに）。
**状態** expanded / collapsed をシェブロン回転＋高さで示す。hover はトリガ行のみ `--c-sunk`。
**規則** 07 抑制: 既定で重要情報を畳んで隠さない（判断材料を初期から見せる, `M-JUDGMENT-MATERIAL`）。装飾アニメは付けず開閉の因果だけ短く。Alexander: FAQ 等の反復は Accordion パターンに統一。開閉を色だけでなく形（シェブロン）で伝える。
**a11y** トリガは `<button aria-expanded>` ＋ `aria-controls`。パネル内に見出し階層 `<h3>` を残す。`prefers-reduced-motion` で高さアニメを抑制。キーボードで開閉可能に。
**関連** Disclosure／Tabs（切替）／Tree view。

### Heading

**用途** セクションの導入。サイズ・ウェイトで重要度と文書階層を示し、読み順を決める。
**変種** H1（ページに1つ）〜H6 / overline（区分ラベル）/ 見出し＋補足（eyebrow・副題）。
**構成** 見出しテキスト（＋任意の overline・補足）。装飾下線・色面で階層を作らない。
**寸法** type scale 限定段: H1 28/600、H2 22/600、H3 18/600（06）。見出し下は近く（8–12px）、前のブロックとは遠く（spacing-8 32px）＝直後の内容に属させる。
**色** `--c-ink`（副次は `--c-ink-soft`）。色で重要度を作らない＝サイズ・ウェイト差で作る。
**状態** 原則状態なし（インタラクティブ化しない）。
**規則** 06 / 02: size 順＝weight 順＝意味の重要度順を一致させ逆転させない。階層を飛ばさない（H1→H3 と段を抜かない）。装飾目的のサイズ変更を禁止（目立たせは余白で）。1画面の最上位 H1 は1つ。
**a11y** 見た目のサイズと無関係に `<h1>`〜`<h6>` を文書順で入れ子にする（スキップ禁止）。スタイルは class、レベルは意味で決める。スクリーンリーダの見出しジャンプの目印になる。
**関連** Typography（06）／Stack／Separator。

### Avatar

**用途** ユーザー・エンティティを小さな図像で識別する。リスト・コメント・ヘッダ等で「誰か」を示す。
**変種** 画像 / 初期文字（イニシャル）フォールバック / アイコン / ステータスドット付き / グループ（重なり表示）。
**構成** 円または角丸方形の枠＋画像 or イニシャル。任意で在席ドット・バッジ。
**寸法** 一般 24 / 32 / 40px、サイズはスケールから選ぶ。円形は正方比で3–5%オーバーシュート（円が小さく見える錯視補正, opticalAdjustment.circleOvershoot 1.05）。
**色** 画像欠落時のイニシャル地は `--c-sunk`＋文字 `--c-muted`（蛍光色のランダム背景にしない）。枠が要るなら `--c-line`。
**状態** 画像ロード失敗→イニシャル・アイコンへフォールバック（壊れ画像を出さない）。
**規則** 07: 装飾でなく識別が目的。ステータスは色だけでなくドット位置＋ラベルで伝える（色のみ依存禁止, 10）。Alexander: イニシャル生成規則を共通化し one-off にしない。
**a11y** 意味を持つ画像 avatar は `alt="氏名"`、純装飾なら `alt=""`。ステータスドットは `aria-label` か visually hidden テキストで状態語を補う。
**関連** Image／Badge／List。

### Image  〔Picture〕

**用途** 写真・図版・スクリーンショットを埋め込む。情報を運ぶ図か、純装飾かを最初に区別する。
**変種** 内容画像 / 装飾画像 / レスポンシブ（`<picture>`・`srcset`）/ 図＋キャプション（`figure`）/ アスペクト固定サムネ。
**構成** 画像本体（＋任意 `figcaption`）。容器でアスペクト比を固定し `object-fit: cover`。
**寸法** 角丸はカード内なら `R_inner = R_outer − padding`、単体サムネは golden-2 6px。比率は枠で予約（CLS 防止）。
**色** プレースホルダ・読込中地は `--c-sunk`（スケルトン）。枠線が要るなら `--c-line`、影で浮かせない。
**状態** 読込中スケルトン→表示、失敗時は代替テキスト・アイコン。遅延読み込みで下方画像を後追い。
**規則** Tufte: 意味のない装飾画像を敷かない（chartjunk）。比較する図は同一スケール・同一トリミングで並べる。01: 題材固有の図（実測図・図面等）を「それらしい型」で代替しない。`width/height` か `aspect-ratio` を必ず指定。
**a11y** 内容画像は内容を述べる `alt`、装飾は `alt=""`（読み上げから除外）。図表は `<figure>＋<figcaption>`。文字を焼き込んだ画像を避ける。
**関連** Avatar／Card／Video／Carousel。

### Quote  〔Blockquote〕

**用途** 引用文・証言・プルクオートを本文と区別して示す。出典を伴う。
**変種** ブロック引用（`blockquote`）/ プルクオート（本文からの抜き出し強調）/ 証言（顧客の声）/ インライン引用（`q`）。
**構成** 引用本文＋出典（`cite`：氏名・媒体）。任意でアバター。装飾の巨大引用符を主役にしない。
**寸法** 左インデント＋ヘアラインの縦罫（`border-left` `--c-line`）で引用範囲を示す。padding 縦<横。本文は読みやすい行長（和文40–50字）。
**色** 縦罫 `--c-line`／本文 `--c-ink`、出典 `--c-muted`。地色面を敷くなら `--c-sunk` 程度に抑える。
**状態** 原則状態なし。
**規則** 07 / 01: 中央寄せの装飾的大型引用を既定にしない（生成ランディングの定番骨格＝テンプレ顔になる, 01）。引用は左揃え・本文と地続きの読み順に置く。Tufte: 引用符画像や飾り罫を足さず、インデントと出典だけで成立させる。出典のない「それっぽい引用」を作らない（誠実）。
**a11y** ブロック引用は `<blockquote>`、出典は `<cite>`、インラインは `<q>`。CSS で足す引用符は読み上げに混ぜない。
**関連** Card（証言カード）／Separator／Typography。

### Separator  〔Divider〕

**用途** 意味的に異なる群の境界を1本の線で示す。余白で足りるなら引かない。
**変種** 水平 / 垂直（インライン区切り）/ ラベル付き（「または」等）/ メニュー内グループ区切り。
**構成** 1px のヘアライン。ラベル付きは線＋中央テキスト。
**寸法** 太さ1px、角丸なし。前後に群間余白（spacing-6 24px〜spacing-8 32px）を確保し、線だけで詰めない。
**色** `--c-line`（強調が要る場合のみ `--c-line-strong`）。濃い実線・影付き線にしない。
**状態** 状態なし（純粋な構造要素）。
**規則** 04 / 07: 近接（余白差）で群を分けられるなら線は不要＝削減テストで消す。線は「余白だけでは境界が読めない」ときの最小手段。Tufte: data-ink を増やすだけの装飾罫を足さない。連続して何本も引かない（線の量産は箱化と同じノイズ）。
**a11y** 意味上の区切りなら `<hr>`（暗黙 role=separator）、純装飾なら `role="presentation"` か CSS ボーダーで。スクリーンリーダに無意味な区切りを多発させない。
**関連** Stack／List／Heading。

### Stack

**用途** 子要素に一定の余白（リズム）を与える縦・横の容器。間隔をトークンで統一する仕組み。
**変種** 縦スタック（VStack）/ 横スタック（HStack）/ 区切り線入り / 折返し（wrap）。
**構成** 容器＋子要素。間隔は `gap`（または隣接マージン）で一括管理し、子に個別マージンを散らさない。
**寸法** `gap` は spacing トークンから1つ選ぶ（群内 8px / 関連 16px / 群間 24–32px）。要素内＜要素間＜セクション間の階層を守る。
**色** 容器自体は透明が基本（面が要るときだけ `--c-surface`）。
**状態** 状態なし（レイアウトプリミティブ）。
**規則** 04 / 06: Visual Rhythm をトークンで作る＝半端値（17 / 23px）を混ぜない。Alexander: 余白を毎回手で指定せず Stack パターンに集約し one-off を排除。近接の原則＝同一 Stack 内は一群、Stack をネストして群間に段差を作る。中央寄せに逃げず左・上揃えを既定に（04）。
**a11y** Stack は視覚的余白のみで、意味の区切りには Heading / Separator を併用。DOM 順＝読み順＝フォーカス順を一致させる（視覚順を CSS で入替えない）。
**関連** Separator／List／全レイアウト。

### Hero  〔Jumbotron〕

**用途** ページ先頭で題材の最も特徴的な主張を最初に置く帯。型ではなく主張。
**変種** 題材により様式が変わる（実測図・目録・票・記事リード等）。汎用の「eyebrow＋大見出し＋2ボタン＋右図版」を既定にしない。
**構成** 題材固有の主役要素＋必要最小の導線。各単位が「何の情報を運ぶか」を言えること。
**寸法** バナー系 padding `32px 48px`（1:1.5, asymmetricPadding.banner）。見出しは H1 段。中央寄せは短い単独見出しに限定し、本文・リストは左揃え。
**色** 面 `--c-surface` / `--c-canvas`。グラデ地・強い影で持ち上げない（07）。アクセントはプライマリ1色に絞る。
**状態** 原則状態なし（CTA を含む場合はその Button 仕様に従う）。
**規則** 01（最重要）: 汎用ランディング骨格を借りない＝題材の native な様式から構造を導く。テンプレ骨格は中を抑制しても「どこかで見た型」を発する。装飾的 01/02 番号・中央引用・等分3カラム特徴の定番連結を避ける。全要素が情報を運ぶ（装飾だけの帯を置かない）。
**a11y** 見出しは `<h1>`（ページに1つ）。背景画像に文字を焼き込まず HTML テキストで。CTA はキーボード到達可能に。
**関連** Heading／Image／Button／Card。

### Carousel  〔Slider〕

**用途** 限られた幅で複数スライドを順に見せる。横並びで収まらない関連メディア群。
**変種** 画像カルーセル / カードカルーセル / 1枚ずつ / ピーク表示（次が少し見える）。
**構成** スライドトラック＋前後ボタン＋位置インジケータ（ドット・番号）＋（任意）サムネ。
**寸法** スライド間 gap はトークン。操作ボタンは48px、インジケータ間隔8px。角丸はメディアに合わせ golden-2 / 1。
**色** 操作（矢印・ドット）を `--c-ink` / `--c-muted`、現在位置を `--c-accent`。淡い矢印を背景に埋もれさせない。
**状態** 現在スライド・総数を常時表示（位置の喪失を防ぐ）。矢印は hover で初めて出さず常時可視。自動再生は既定でオフ。
**規則** 09 / 07: 自動再生・無限ループ装飾を避ける（動きは因果を伝えるときだけ, `prefers-reduced-motion` で停止）。Tufte / 01: 重要情報をスライドで隠さない＝最初の画面で判断材料を見せる。比較対象を別スライドに分けず同一軸で並置できないか先に検討。Alexander: 1パターンに統一。
**a11y** 自動再生時は一時停止コントロールを必ず提供。`aria-roledescription="carousel"`／各スライドにラベル。矢印・ドットは `<button>`＋ラベル。キーボードで前後移動。
**関連** Image／Card／Pagination／Tabs。

### Video

**用途** 動画を再生する。説明・デモ・記録。自動再生・音の扱いを最初に決める。
**変種** プレーヤ（コントロール付き）/ 背景動画（装飾）/ 外部埋め込み / ポスター画像付き。
**構成** `<video>`＋ネイティブ・カスタムコントロール＋ポスター＋字幕トラック（`<track>`）。
**寸法** アスペクト比を枠で固定（16:9 等, CLS 防止）。コントロールのタッチ44–48px。角丸はメディアに合わせる。
**色** コントロール面は内容より目立たせない。プレースホルダ・読込は `--c-sunk`。
**状態** 読込中→再生可、バッファリング、再生 / 一時停止、エラー（再試行導線）。自動再生するなら必ずミュート＋一時停止可。
**規則** 09 / 10: 装飾の背景動画・自動再生を多用しない（`prefers-reduced-motion` を尊重し動きを止める）。07: 中身のない演出にしない（実際の内容を運ぶ）。誠実＝偽の再生 UI を出さない。
**a11y** キャプション・字幕（`<track kind="captions">`）を提供。コントロールをキーボード操作可能に。自動再生＋音を既定にしない。`aria-label` でプレーヤを説明。
**関連** Image／Carousel／File。

### File  〔Attachment〕

**用途** 添付・ダウンロード対象のファイルを1要素で表す。種類・サイズ・操作を一目で示す。
**変種** 添付チップ / ダウンロード行 / アップロード済みプレビュー / 画像サムネ付き / 進捗付き。
**構成** 種別アイコン（拡張子）＋ファイル名＋メタ（サイズ・更新日）＋操作（DL / 削除 / 再試行）。
**寸法** 行は最小高44px、padding 縦<横。角丸 golden-2 6px。アイコンとテキストは近接（一組として読ませる）。
**色** 面 `--c-surface`＋ヘアライン `--c-line`。名 `--c-ink`／メタ `--c-quiet`（小さい本文には使わずメタのみ）。種別色は控えめに。
**状態** アップロード中は進捗（実際の進捗を表示, 偽進捗禁止）、完了、エラー（理由＋再試行）、権限なし。
**規則** Tufte / 02: サイズ・種別は判断材料＝省略しない。Alexander: ファイル種別→アイコンの対応を共通化し one-off にしない。08: 失敗状態（容量超過・形式不正）に回復導線。アイコンだけで種別を伝えず拡張子テキストも併記（色・記号のみ依存回避）。
**a11y** 操作はネイティブ `<button>` / `<a download>`。アイコンに種別ラベル（`aria-label` か visually hidden）。省略表示でも完全名を `title` ・読み上げに残す。
**関連** File upload（forms）／List／Image。

### Visually hidden  〔Screenreader only〕

**用途** 視覚的に隠しつつ支援技術には伝えるユーティリティ。アイコンのみボタンのラベル、文脈補足、見出し補完等。
**変種** 常時不可視（読み上げのみ）/ `focus` 時のみ可視化（Skip link 用）。`hidden` / `display:none`（AT からも消える）とは別物。
**構成** clip した1px の要素。`display:none` / `visibility:hidden` を使わない（読み上げから消えるため）。
**寸法** 標準パターン: `position:absolute; width:1px; height:1px; clip-path:inset(50%); overflow:hidden; white-space:nowrap;`。
**色** 不可視のため配色なし。
**状態** Skip link 等では `:focus` で可視に戻し、フォーカス位置を見せる。
**規則** 10: 冗長符号化の担い手＝アイコンのみの操作・色のみの状態に「成功」「閉じる」等のテキストを補う。乱用しない（見せるべき情報を隠す代替にしない）。装飾アイコンには付けず `aria-hidden` 側で消す。
**a11y** 中核用途。`<table>` の補足見出し、アイコンボタンのラベル、フォーム補助説明、ライブ領域の文言に使う。`aria-hidden="true"` と併用しない（矛盾）。
**関連** Skip link／Icon／Label／全アイコンボタン。
