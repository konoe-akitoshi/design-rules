# 07 — Restraint & Form（抑制と造形）

造形を機能から導き、削れるものを削る。装飾で機能を誇張せず、見た目を実装と一致させる。
関心: **誠実さ・道具性**。決定順の段6。きれいさは目的ではなく、作業対象を前に出した結果である。

各項目は `思想 → その思想を使うための理論 → 使える規則` の3層。

---

## Dieter Rams — 可能な限り少なく、対象を前に出す

**思想**
良いデザインは控えめで誠実で長持ちする。可能な限り少ないデザインで、機能を実際以上に見せず、流行に依存しない。

**その思想を使うための理論**
画面のすべての要素は **信号**（理解・操作に必要なもの）か **ノイズ**（装飾・自己主張）のどちらか。装飾を足すほど信号対雑音比（signal-to-noise）が下がり、本当に注目すべきものが埋もれる。とりわけ UI クローム（影・枠・背景・自己主張する装飾）自身が内容より目立てば、それ自体が大きなノイズになり、作業対象を後ろへ退ける（VIS-03 unobtrusive）。だから「可能な限り少なく」は美学ではなく、信号を濃く保つための工学的要請である。
運用は要素ごとの**削減テスト** — この要素を消したら、ユーザーは何ができなくなるか？を問う。答えが「何も困らない（見た目が寂しくなるだけ）」なら**装飾＝削る**。「押せる／今どの状態か／次に何をするかが分からなくなる」なら**手がかり＝残す**。この装飾／手がかりの二分が削減の判断基準であり、ミニマル化の暴走を止める線でもある（手がかりは削らない＝メタ下限 VIS-05）。
誠実さは「機能を実際以上に見せない」こと。過剰な影で平らな要素を浮かせる、押せないものを押せそうに見せる、何も起きていないのにローディング演出を出す — これらはユーザーの期待と実装をずらす嘘であり、信頼を削る。長命性は「流行に寿命を預けない」こと。トレンド由来の装飾は寿命が短く、流行が剥がれると古く見える。構造と操作に根ざした形だけが長く保つ。

**使える規則**
- 要素ごとに削減テストをする。消して困らない装飾は削り、手がかり（affordance・状態・ラベル）は残す（VIS-05）。
- UI クローム（影・枠・背景）を内容より目立たせない。装飾過多なヒーロー・自己主張する装飾を避け、コンテンツファーストのミニマルなレイアウトにする（VIS-03）。
- `shadow-lg` `shadow-xl` 等の強い影を使わない。`shadow-sm` かヘアライン境界（`border` + `--c-line`）で十分。
- グラデーション背景を多用しない。ソリッドカラーを基本にする。
- カード上部のカラーバー（`border-t-4 border-blue-500` 等）を使わない。全周ボーダーまたは装飾なしにする。
- 機能を実際以上に見せる誇張（浮かせる影・偽の操作感・中身のない演出）を避ける（VIS-04 / 誠実）。
- 流行の装飾に寿命を預けない。構造・操作に根ざした造形を選ぶ（長命）。

## Bauhaus — 形は機能と素材から導く

**思想**
形は任意の装飾ではなく、用途・素材・構造・制作方法から導かれる。見た目・機能・実装・制約は同じ設計問題の異なる側面で、分離しない。

**その思想を使うための理論**
form follows function/material は「すべての造形（サイズ・形・配置・角丸・影）に、用途か制約からの根拠がある」という制約として効く。各造形に「なぜこの形か」を問えば、根拠を説明できる信号と、説明できないノイズが分かれる — 根拠なき造形は削除対象（VIS-01）。素材（material）も根拠になる。入力欄は「書き込める面」という素材性を見せ、ボタンは押し込める塊として見せる。形が素材の振る舞いを正しく告知することが、用途からの導出の半分を担う。
Rams の誠実を、Bauhaus は実装レベルで言い直す — **見た目は実装と一致しなければならない**。押せそうに見えて押せない、影で浮いて見えるが実際は固定、進捗バーが動くが裏では何も起きていない、はすべて「見せかけの形」＝嘘であり、ユーザーの概念モデルを壊す（VIS-02）。だから造形は機能を**約束**するものとして扱い、約束した機能を実際に持たせる。
そして優先順位は明確 — **道具的完成度**（速い・正確・壊れない・読める）を**装飾的完成度**（きれい）より先に満たす。きれいさは道具が成立した後の仕上げであって、入口ではない。形から入って機能を後付けすると、用途に合わない造形が固定され、後から剥がせなくなる。

**使える規則**
- すべての造形（サイズ・形・配置・角丸・影）に、用途か制約からの根拠を持たせる。説明できない造形は削る（VIS-01）。
- 見た目と実装を一致させる。押せる見た目は実際に押せる、浮く影は階層を表す、進捗表示は実際の進捗を表す（VIS-02 / 見せかけの形を作らない）。
- 角丸・余白・サイズも制約から導く。入れ子角丸は `R_inner = R_outer − padding` のように構造関係を造形へ反映する（VIS-01 / [values/border-radius.md](./values/border-radius.md)）。
- 形を先に決めて機能を後付けしない。用途 → 制約 → 形の順で確定する。
- 「道具として成立しているか（速い・正確・読める）」を、きれいさより先に確かめる。

## Linear — 静かで速い作業道具

**思想**
過剰装飾を避けて作業対象を前に出し、状態を常時可視化し、反復操作を速くする。キーボードを前提にした、静かで速い作業道具。Quiet Interface は Rams の抑制の現代 SaaS 版。

**その思想を使うための理論**
反復作業の道具（開発者向け SaaS・管理画面・ダッシュボード）では、視覚表現と操作速度が矛盾してはならない。過剰装飾は二重に害をなす — 信号を薄め（Rams）、かつ描画と操作を重くする。だからクローム（枠・背景・装飾）を抑えて作業対象を主役にする（Quiet Interface＝Deference）。
ただし静けさは情報不足の言い訳ではない。状態（同期中・保存済・エラー・選択中）は常時可視化し、Calm の周辺表示として控えめに出す（静か ≠ 不明瞭）。「美しいが遅い／静かだが情報が足りない／密度は高いが読めない」のどれも避けるのが Linear の参照点たるゆえん。
熟練者の速度を支配するのはアクセスコスト（クリック数・移動距離・階層の深さ）なので、反復操作は最短化し、キーボード経路（ショートカット・コマンドパレット）を前提に置く（VIS-08）。動きは状態変化・因果を伝えるときだけ短く使い、装飾アニメは置かない（VIS-09）。要するに Linear は、Rams の抑制・Raskin の操作速度・Calm の静けさを統合し、現代の作業道具として具体化した到達点である。

**使える規則**
- クロームを抑え、作業対象を前に出す。装飾より状態と密度を優先する（Quiet Interface）。
- 状態を常時可視化する（同期・保存・エラー・選択を、控えめなインジケータ・ambient 表示で）。
- 反復操作のアクセスコストを最小化し、主要操作にショートカット／コマンドパレットを用意する（VIS-08）。
- 余白で密度を薄めず、関係を分けるために使う。静けさを理由に判断材料（現在値・状態・差分）を削らない（`M-JUDGMENT-MATERIAL`）。
- アニメーションは状態変化・因果を伝えるときだけ短く自然に使う。装飾アニメ・常時アニメを避ける（VIS-09）。
- 絵文字を多用しない。UI アイコンには SVG を使う。
- `text-black` を直接使わず `--c-ink`（slate 系濃色）を使う。1ページのアクセントはプライマリ1色＋セマンティック（赤/緑/黄）に絞る。

## 統合 — この章の使える規則（チェックリスト）

1. 要素ごとに削減テストをしたか — 消して困らない**装飾は削り**、**手がかり**（affordance・状態・ラベル）は残したか（装飾／手がかりの二分, VIS-05）
2. UI クローム（影・枠・背景）が内容より目立っていないか／装飾過多なヒーローになっていないか（VIS-03 unobtrusive）
3. 影は `shadow-sm` 以下か／グラデーションを多用していないか／カード上部のカラーバー（`border-t-4`）がないか
4. 機能を実際以上に見せる誇張（浮かせる影・偽の操作感・中身のない演出）がないか（VIS-04 / 誠実）
5. すべての造形（サイズ・形・配置・角丸・影）に、用途・制約からの根拠があるか（VIS-01）
6. 見た目と実装が一致しているか — 押せる見た目は押せる、進捗は実際の進捗か（見せかけの形がないか, VIS-02）
7. 状態が常時可視化されているか／反復操作がキーボードで速いか（VIS-08）
8. アニメーションは状態変化・因果を伝えるときだけ短いか（VIS-09）
9. `text-black` を避け `--c-ink` を使い、アクセントをプライマリ1色＋セマンティックに絞ったか／絵文字を多用していないか

衝突時の優先: **手がかりはメタ下限**。抑制・ミニマル・静けさを理由に、手がかり（affordance・状態・ラベル・判断材料）を削らない（VIS-05 / `M-JUDGMENT-MATERIAL`）。そして道具的完成度をきれいさより先に満たす。
