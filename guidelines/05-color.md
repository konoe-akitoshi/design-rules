# 05 — Color（色の使い方）

抑制された配色で、状態と階層を伝える。
関心: **抑制・注意の階層・色の data-ink・WCAG/色覚**。決定順の段5（造形）。
具体的なパレットは [color-system](./values/color-system.md) が定義済み — この章はその「**どう使うか**」の層。hex を再定義せず、`--c-*` トークンと役割を参照する。

各項目は `思想 → その思想を使うための理論 → 使える規則` の3層。

---

## Dieter Rams — 色は内容に従属する

**思想**
良いデザインは控えめ。UI のクローム（色・装飾）は作業対象に従属し、UI 自身は必要な時だけ働く。

**その思想を使うための理論**
色は最も強い視覚的注意の合図。だから UI が色で自己主張すると、内容（ユーザーの作業対象）より目立ち、内容への従属（Deference）が崩れる。抑制の運用は2手 — (1) ベースを無彩色（slate ランプ）に寄せ、彩度の高い色は **アクセント1色＋セマンティック**だけに予算化する（多色は統一感を壊す）。(2) 面の分離は影でなく**ヘアライン境界**で行う。強い影は光源の嘘で誠実さ（Honesty）に反し、過剰装飾になりやすい。色数を減らすことは情報を削ることではなく、注意の合図を整理することである。

**使える規則**
- ベース面は無彩色（`--c-canvas` / `--c-surface` / `--c-sunk`）、テキストは `--c-ink` / `--c-muted`。`#000` / `text-black` の直接使用は禁止（→ `--c-ink`）。
- 彩度の高い色はアクセント1色（`--c-accent`）＋セマンティックに限定。1画面に3色以上の*意味*のアクセントを散らさない。
- 面の分離はヘアライン境界（`--c-line` / `--c-line-strong`）で。`shadow-lg` / `shadow-xl`・グラデーション地で面を持ち上げない。
- カード上部のカラーバー（`border-t-4` 等）禁止。全周ボーダーまたは装飾なしにする。

## Calm Technology — 色は有限の注意予算

**思想**
技術は平常時は静かに周辺にあり、必要な時だけ前景化する。注意は有限資源。

**その思想を使うための理論**
彩度・コントラストは注意を引く強さそのもの。すべてを色で強調すると優先順位が消え（全部色付き＝何も目立たない）、本当に変化のあった対象が埋もれる。だから色を「予算」として配り、平常状態は無彩色で背景化し、強調（アクセント／セマンティック地）は **変化のあった対象・いま操作している対象だけ**に使う。常時点灯の赤バッジや常時アニメする色は予算の浪費。状態色の強さも重要度順に配分する。ただし「静か ≠ 不明瞭」— 静けさを理由に重大なエラー・期限の色を弱めてはならない（メタ下限 `M-CRITICAL-VISIBILITY`）。

**使える規則**
- 平常状態は無彩色で背景化する。アクセント・セマンティック地は変化／現在の対象にだけ使う。
- 常時点灯の赤バッジ・常時アニメーションする色を避ける（注意予算の浪費）。
- 状態色の強さは重要度順（danger > warning > success > info / neutral）。すべてを同じ強さで塗らない。
- それでも重大なエラー・期限の色は確実に視認できる強さを保つ（静けさを理由に薄めない）。

## Edward Tufte — 色は分類・状態を伝えるための data-ink

**思想**
データそのものを前に出し、無意味な装飾インクを減らす。色も例外ではない。

**その思想を使うための理論**
色分けは強力な分類の合図だが、意味に対応しない色分けは chartjunk — 読み手は存在しない区別を探し、認知負荷だけが増える。だから色は **分類軸・状態・強調**を伝えるときだけ使い、装飾目的の色分け（行ごとに違う色、意味のないカテゴリ色）を排除する。原則は色とエンコーディングの一対一 — 同じ意味は同じ色、違う色は必ず違う意味を持つ。逆に、抑制やミニマル化を理由に判断材料の色（しきい値超過・差分・現在状態）まで削ってはならない。装飾の色は減らし、意味の色は残す（メタ下限 `M-JUDGMENT-MATERIAL`）。

**使える規則**
- 色は分類・状態・強調を伝えるときだけ使う。装飾目的の色分けを排除する。
- 同じ意味は同じトークン、違うトークンは違う意味（色とエンコーディングを一対一に保つ）。
- 状態・凡例は color-system のセマンティック役割（success / warning / danger / info）に揃え、独自色を増やさない。
- 抑制を理由に、判断に必要な色（しきい値超過・差分・現在状態）まで削らない。

## WCAG／色覚 — 誰の目にも読める配色

**思想**
配色は美しさより先に「読めること」を満たす。色の知覚は人によって異なる。

**その思想を使うための理論**
コントラスト比は可読性の物理的な下限で、満たさない配色は一部のユーザーに文字が消える（メタ下限 `M-LEGIBILITY`）。基準は本文 4.5:1・大文字/UI 部品 3:1（`tokens.json` の `wcag`）。さらに色覚多様性により、色相だけの区別（赤×緑など補色の直接配置）は一部の人に判別できない。だから状態は **色だけで伝えず**、アイコン・ラベル・テキストで二重化し（WCAG 1.4.1）、区別は色相でなく明度差で担保する。color-system のセマンティックが「深く静かな」低彩度で蛍光色を避けるのは、明度差を確保しつつ騒がしさを抑えるためである。focus も色のみに頼らず明示的なリングで示す。

**使える規則**
- コントラスト: 本文は `--c-ink` / `--c-muted` で 4.5:1、UI 部品・大文字は 3:1 を確保。`--c-quiet` を小さい本文に使わない。
- 状態を色だけで伝えない。アイコン＋ラベルを必ず併用する（色覚多様性・WCAG 1.4.1）。
- 補色（赤×緑など）を文字×背景で直接配置しない。区別は明度差で担保する。
- focus は `outline: 2px solid var(--c-accent); outline-offset: 1px` で必ず可視化する（色だけに頼らない）。

## 役割と状態のマッピング（hex は [color-system](./values/color-system.md) を正とする）

役割は色でなくトークンで指定する。この章はどの役割にどのトークンを割り付けるかだけを扱い、値は持たない。

| 役割 | トークン |
|------|---------|
| 背景・面 | `--c-canvas`（ページ）/ `--c-surface`（カード）/ `--c-sunk`（沈み・hover 地） |
| 境界（影の代わり） | `--c-line`（通常）/ `--c-line-strong`（強調） |
| テキスト | `--c-ink`（見出し・本文）/ `--c-muted`（説明）/ `--c-quiet`（ラベル・補助、本文不可） |
| アクセント | `--c-accent`（主要操作・リンク・現在地）/ `--c-accent-ink`（hover・押下）/ `--c-accent-sunk`（選択地） |
| セマンティック | success / warning / danger / info の `text` / `bg` / `border` 3役割 |

| 状態 | 表現 |
|------|------|
| hover | 面 `surface → sunk` / アクセント `accent → accent-ink` |
| active・選択中 | 地 `accent-sunk` ＋ 文字 `accent` ＋ 境界 `line` |
| disabled | 文字 `quiet` ＋ 地 `sunk`（彩度を上げない） |
| focus | `outline: 2px solid var(--c-accent)`（必ず可視） |

## 統合 — この章の使える規則（チェックリスト）

1. ベース面・テキストが無彩色トークンか（`#000` / `text-black` を直接使っていないか）
2. 彩度の高い色がアクセント1色＋セマンティックに絞られているか（3色以上の意味で散らしていないか）
3. 面の分離がヘアライン境界か（`shadow-lg` / `shadow-xl`・カラーバー・グラデーションで持ち上げていないか）
4. 平常は無彩色で、強調が変化／現在の対象に絞られているか（常時赤バッジ・常時アニメ色がないか）
5. 色分けが意味（分類・状態・強調）に対応しているか（装飾目的の色分けがないか／同じ意味＝同じトークンか）
6. 本文 4.5:1・UI/大文字 3:1 を満たすか（`--c-quiet` を小さい本文に使っていないか）
7. 状態を色だけで伝えず、アイコン・ラベルを併用しているか（補色の直接配置がないか）
8. focus が `--c-accent` で必ず可視化されているか

衝突時の優先: 可読性(6) と 判断材料(5) はメタ下限。抑制・静けさより、コントラスト下限（`M-LEGIBILITY`）と判断に必要な色（`M-JUDGMENT-MATERIAL`）を優先する。さらに重大変化の色は静けさより優先（`M-CRITICAL-VISIBILITY`）。
